🔍 Panoramica

Il plugin os-upnp per OPNsense fornisce funzionalità Universal Plug and Play (UPnP) utilizzando miniupnpd. A partire dalla versione 17.1 di OPNsense, il servizio UPnP è stato trasferito dal core a un plugin individuale e deve essere reinstallato come plugin per essere utilizzato.

⚠️ Importante: Non esiste una documentazione ufficiale specifica per il plugin os-upnp sul sito docs.opnsense.org. La maggior parte delle informazioni proviene da discussioni della community e forum.

🚀 Procedura di Installazione

1. Navigare in System → Firmware → Plugins nell'interfaccia web OPNsense 2. Cercare "os-upnp" nella barra di ricerca della colonna Name 3. Cliccare sul pulsante Install 4. Si verrà reindirizzati alla pagina Updates e il plugin verrà installato

⚙️ Configurazione Base

1. Abilitare il Servizio: Services > Universal Plug and Play 2. Disabilitare IGMP Snooping sulle reti client 3. Configurazione Sicurezza: Impostare UPnP su default-deny 4. Aggiungere Regole Allow: es. "allow 1024-65535 192.168.1.10 1024-65535"

🔒 Considerazioni di Sicurezza

Importante: UPnP è intrinsecamente insicuro. Qualsiasi programma sulla rete può consentire e inoltrare qualsiasi traffico. È un classico esempio del compromesso "Sicurezza vs. Convenienza".

🔧 Porte Utilizzate da miniupnpd

  • UDP 1900: Simple Service Discovery Protocol (SSDP)
  • UDP 5351: PCP (Port Control Protocol)
  • TCP 2189: HTTP e query SOAP

🎮 Configurazione per Gaming

Per il gaming con NAT stretto, è necessario:

  • Utilizzare mappature porta statiche (Static Port Mapping)
  • Configurare Hybrid NAT per il minor impatto sulla sicurezza
  • Aggiungere regole firewall specifiche se necessario

⚠️ Problemi Noti (2024)

  • UPnP può impostare correttamente le mappature porta ma non creare regole NAT per il routing del traffico
  • Alcuni problemi con la versione 1.5_5 che impedisce l'aggiunta di port forward
  • Creazione di regole rdr oltre l'ambito previsto in alcune configurazioni

🔍 Troubleshooting

Per il debug:

  • Controllare i log del firewall in Status > System Logs > Firewall
  • Verificare lo stato UPnP in Status > Universal Plug and Play
  • Assicurarsi che le regole firewall permettano il traffico UDP sulla porta 1900
